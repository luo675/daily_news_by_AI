# 项目总览

## 1. 项目定位

`daily_news` 是一个面向 AI 领域的个人知识数据库项目。

它的目标不是做一个普通的资讯抓取器，而是建立一套可持续积累、可检索、可关联、可供通用 AI Agent 读取的知识系统，用于支持：

- 每日简报生成
- 专题串联
- 投资研究辅助
- 产品机会判断

第一阶段的首要目标是产品机会判断。

## 2. 项目目标

项目要解决的问题：

- AI 领域高价值信息分散在博客、访谈、演讲、播客和行业讨论中
- 原始信息噪声大，不适合直接用于判断
- 通用 AI 缺少经过整理的专用知识上下文
- 同一主题跨时间演化，难以人工长期跟踪

项目希望建立以下能力：

- 持续采集高价值 AI 信息
- 将内容处理为摘要、标签、主题、实体、关系
- 保存结构化处理结果
- 生成中英双语日报
- 生成结构化产品机会判断结果
- 对外提供给通用 AI Agent 调用的结构化 API

## 3. 核心用户

第一阶段核心用户：

- 项目拥有者本人

后续传播对象：

- 其他研究者
- 投资分析者
- 创业者与产品经理
- 可调用知识库 API 的通用 AI Agent

## 4. 第一阶段范围

第一阶段包含：

- 英文优先的数据采集
- 技术博客、演讲、访谈、文字稿处理
- 清洗、去重、分段、摘要、标签、实体、主题抽取
- 来源可信度分级
- 冲突检测与不确定性标记
- 产品机会评分
- 中英双语日报
- 手动关注列表
- 人工修订入口
- 对外结构化 API

第一阶段不包含：

- 多人权限系统
- 复杂图谱可视化
- 全量网页快照归档
- 音视频转录主流程
- 完整公开分发平台

## 5. 数据源策略

语言策略：

- 英文优先
- 输出阶段采用中英双语并列

第一阶段优先来源类型：

- 技术博客
- 演讲文字稿
- 访谈文字稿
- 播客文字稿

推荐的 P0 来源样板：

- 技术博客与研究者长文：Andrej Karpathy、Ethan Mollick、李飞飞相关公开文字内容、Jim Fan 公开文字内容
- 高价值访谈与深度对谈文字稿：Dwarkesh Podcast、a16z Podcast、20VC、The Robot Brains Podcast
- 投资与创业视角节目文字稿：All-In Podcast、Interplay VC Podcast、TechCrunch Equity Podcast、TBPN

## 6. 产品机会判断目标

第一阶段重点是从信息中发现：

- 真实存在的需求
- 尚未被充分满足的市场空白
- 可产品化的方向
- 值得继续跟踪的机会窗口

默认评分维度：

- 需求真实性：30%
- 市场空白度：30%
- 产品化可行性：20%
- 跟进优先级：10%
- 证据充分度：10%

评分采用 `1-10` 分制，`10` 分代表最有创意、最令人眼前一亮。

## 7. 当前关键决策

- 第一阶段按个人使用设计
- 后续会分享成果给其他人
- 第一阶段主目标是产品机会判断
- 日报每天固定生成一份，支持按需额外生成
- 第一阶段先手动触发更新，后续再自动化
- 冲突信息优先自动判断，判断不了则保留不确定性
- API 第一阶段全部开放，但需要单用户密钥和配额控制
- 关注列表支持按对象类型和按优先级分组
