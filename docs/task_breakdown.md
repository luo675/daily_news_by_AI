# 任务拆分

## 1. 任务设计原则

- 任务尽量独立
- 任务应有明确输入和输出
- 任务应能分配给单独模型完成
- 依赖关系尽量简单清晰
- 每个任务都必须有验收标准

## 2. 任务清单

### T1 项目文档骨架

- 目标：建立架构、模型、API、路线图文档框架
- 输入：`goal.md`
- 输出：文档目录与文档初稿结构
- 依赖：无
- 验收标准：文档覆盖项目总览、架构、API、里程碑、测试策略
- 建议交付物：文档目录清单
- 适合模型：通用规划模型

### T2 来源分类与可信度规则

- 目标：定义来源分类、可信度等级和启用标准
- 输入：P0 来源样板、来源类型
- 输出：来源规则表
- 依赖：T1
- 验收标准：每类来源有采集方式、可信度规则、启用标准
- 建议交付物：来源策略文档
- 适合模型：信息建模模型

### T3 领域对象词表

- 目标：定义人物、公司、产品、模型、主题、赛道、关键词对象类型
- 输入：关注对象范围
- 输出：对象类型规范
- 依赖：T1
- 验收标准：对象类型完整且可直接映射到数据模型
- 建议交付物：对象定义表
- 适合模型：信息建模模型

### T4 核心数据模型设计

- 目标：完成表结构和实体关系设计
- 输入：业务目标、来源规则、对象词表
- 输出：数据模型草案
- 依赖：T2, T3
- 验收标准：覆盖文档、摘要、chunk、主题、机会、日报、关注列表、修订记录
- 建议交付物：ER 草案和字段定义
- 适合模型：架构模型

### T5 文档生命周期设计

- 目标：定义文档状态流转
- 输入：数据模型
- 输出：状态机与失败重试策略
- 依赖：T4
- 验收标准：每个状态有进入条件、退出条件、失败处理
- 建议交付物：状态流转表
- 适合模型：流程设计模型

### T6 采集模块设计

- 目标：设计统一采集接口和输入格式
- 输入：来源分类规则、文档状态流转
- 输出：采集模块接口规范
- 依赖：T2, T5
- 验收标准：博客、访谈、文字稿三类来源可映射到统一文档对象
- 建议交付物：采集接口说明
- 适合模型：系统设计模型

### T7 清洗去重模块设计

- 目标：定义清洗和去重规则
- 输入：统一文档格式
- 输出：清洗规则、去重规则、异常规则
- 依赖：T6
- 验收标准：可区分标题重复、链接重复、正文近重复
- 建议交付物：规则清单
- 适合模型：规则设计模型

### T8 摘要与双语输出规则

- 目标：定义摘要字段和中英双语输出规范
- 输入：处理后文档
- 输出：摘要模板、双语模板
- 依赖：T7
- 验收标准：摘要和日报字段使用统一双语规范
- 建议交付物：输出模板
- 适合模型：内容结构模型

### T9 实体与主题抽取设计

- 目标：定义实体和主题抽取结果结构
- 输入：对象词表、处理后文档
- 输出：实体/主题结构定义
- 依赖：T3, T7
- 验收标准：可支撑检索、关注列表和专题聚合
- 建议交付物：抽取 schema
- 适合模型：NLP 结构设计模型

### T10 冲突检测与不确定性规则

- 目标：定义冲突类型和判断机制
- 输入：来源可信度规则、摘要和实体结果
- 输出：冲突规则与不确定性规则
- 依赖：T2, T8, T9
- 验收标准：覆盖事实冲突、观点冲突、时间冲突
- 建议交付物：冲突规则表
- 适合模型：推理规则模型

### T11 产品机会评分框架

- 目标：定义评分维度、权重和输出模板
- 输入：评分目标、冲突规则
- 输出：评分框架
- 依赖：T8, T10
- 验收标准：可生成 `1-10` 分并明确总分计算逻辑
- 建议交付物：评分规则文档
- 适合模型：策略设计模型

### T12 日报模板设计

- 目标：定义固定日报结构
- 输入：摘要规则、评分框架、watchlist 需求
- 输出：日报模板
- 依赖：T8, T11
- 验收标准：包含今日重点、产品机会、趋势、风险、关注更新、待验证问题
- 建议交付物：日报结构模板
- 适合模型：内容结构模型

### T13 关注列表模型设计

- 目标：定义关注项字段、优先级和分组模型
- 输入：关注对象范围
- 输出：watchlist 模型
- 依赖：T3
- 验收标准：同时支持按对象类型和按优先级组织
- 建议交付物：关注列表字段定义
- 适合模型：信息建模模型

### T14 检索接口设计

- 目标：定义关键词、语义、混合检索接口
- 输入：数据模型、实体主题设计、watchlist 设计
- 输出：检索接口草案
- 依赖：T4, T9, T13
- 验收标准：每种检索接口有输入、输出、排序依据
- 建议交付物：接口说明
- 适合模型：API 设计模型

### T15 Agent API 结构设计

- 目标：定义面向 AI Agent 的统一结构化返回格式
- 输入：评分框架、日报模板、检索接口
- 输出：API schema
- 依赖：T11, T12, T14
- 验收标准：返回至少包含 `summary/evidence/opportunities/risks/uncertainties`
- 建议交付物：API schema 文档
- 适合模型：API 设计模型

### T16 API 鉴权与配额设计

- 目标：定义单用户密钥鉴权和配额策略
- 输入：API 草案、配额要求
- 输出：鉴权和配额方案
- 依赖：T15
- 验收标准：说明 key 生命周期、配额扣减维度、未来扩展方式
- 建议交付物：鉴权策略说明
- 适合模型：平台设计模型

### T17 人工修订模块设计

- 目标：定义可修订字段和覆盖规则
- 输入：数据模型、评分框架、日报模板
- 输出：修订模块设计
- 依赖：T4, T11, T12
- 验收标准：支持修订摘要、标签、机会分数、结论、优先级
- 建议交付物：修订字段清单
- 适合模型：后台设计模型

### T18 调度与任务编排设计

- 目标：定义手动触发和后续自动化触发流程
- 输入：采集、处理、日报、API 模块设计
- 输出：任务编排方案
- 依赖：T6, T7, T8, T12, T15
- 验收标准：明确每日固定简报和按需简报的触发链路
- 建议交付物：任务流图
- 适合模型：流程编排模型

### T19 质量评估体系设计

- 目标：定义检索、摘要、评分和冲突处理的质量指标
- 输入：检索、评分、冲突规则
- 输出：质量指标体系
- 依赖：T10, T11, T14
- 验收标准：至少包含摘要质量、检索相关性、机会判断稳定性三类指标
- 建议交付物：质量指标表
- 适合模型：测试策略模型

### T20 第一阶段集成蓝图

- 目标：把全部设计结果整理成实施顺序和依赖图
- 输入：前序任务全部结果
- 输出：MVP 集成蓝图
- 依赖：T4-T19
- 验收标准：可直接作为后续实现阶段排期依据
- 建议交付物：实施蓝图
- 适合模型：项目管理模型

## 3. 任务依赖关系简表

```text
T1 -> T2, T3
T2 + T3 -> T4
T4 -> T5, T14, T17
T2 + T5 -> T6
T6 -> T7
T7 -> T8, T9
T2 + T8 + T9 -> T10
T8 + T10 -> T11
T8 + T11 -> T12
T3 -> T13
T4 + T9 + T13 -> T14
T11 + T12 + T14 -> T15
T15 -> T16
T4 + T11 + T12 -> T17
T6 + T7 + T8 + T12 + T15 -> T18
T10 + T11 + T14 -> T19
T4-T19 -> T20
```

## 4. 推荐执行顺序

1. 先完成 `T1-T5`
2. 再完成 `T6-T13`
3. 再完成 `T14-T19`
4. 最后完成 `T20`
