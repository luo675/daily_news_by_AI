# 测试与质量策略

## 1. 测试目标

第一阶段测试不是为了追求完整自动化覆盖，而是为了确保系统输出对后续判断有足够质量。

重点验证：

- 文档处理结果是否稳定
- 检索结果是否相关
- 冲突判断是否合理
- 产品机会评分是否一致
- API 输出是否稳定、可供 Agent 消费

## 2. 测试分层

### 2.1 单元测试

目标：

- 验证规则逻辑和数据转换逻辑

覆盖范围：

- 来源分类规则
- 去重规则
- 评分公式
- 配额扣减逻辑
- watchlist 组织逻辑

### 2.2 集成测试

目标：

- 验证模块之间的数据流是否打通

覆盖范围：

- 采集到处理
- 处理到入库
- 入库到检索
- 检索到日报
- 检索到机会评分
- 修订到结果覆盖

### 2.3 数据质量测试

目标：

- 验证处理后的结构化结果是否可用

覆盖范围：

- 摘要完整性
- 双语字段完整性
- 实体提取有效性
- 主题归类合理性
- 来源可信度分级正确性

### 2.4 检索质量测试

目标：

- 验证检索召回和排序结果是否有实际价值

覆盖范围：

- query 与结果相关性
- watchlist 加权效果
- 主题检索效果
- 时间范围筛选效果

### 2.5 冲突处理测试

目标：

- 验证冲突识别和不确定性保留机制

覆盖范围：

- 事实冲突
- 观点冲突
- 时间冲突
- 高可信与低可信来源冲突

### 2.6 机会评分稳定性测试

目标：

- 验证评分维度输出是否稳定、一致、可解释

覆盖范围：

- 同类案例是否评分接近
- 高分案例是否确实更有吸引力
- 低证据样本是否会被压低总分
- 不确定性样本是否被正确标记

### 2.7 API 合约测试

目标：

- 验证接口结构稳定，不破坏 Agent 接入

覆盖范围：

- 响应字段完整性
- 必填字段一致性
- 错误响应结构
- 鉴权和配额行为

## 3. 测试样本策略

建议准备固定样本集，覆盖：

- 技术博客样本
- 访谈文字稿样本
- 播客文字稿样本
- 高可信来源样本
- 低可信来源样本
- 明显冲突样本
- 明显高机会样本
- 无明显机会样本

这些样本应放入 `tests/fixtures/` 作为稳定基线。

## 4. 质量指标建议

### 4.1 摘要质量

- 是否覆盖核心观点
- 是否丢失关键限制条件
- 中英内容是否语义一致

### 4.2 检索质量

- Top-N 结果中高相关内容比例
- query 与结果偏题率
- watchlist 相关项命中率

### 4.3 冲突判断质量

- 冲突识别率
- 错误合并率
- 不确定性保留是否合理

### 4.4 机会判断质量

- 高分项是否具有实际吸引力
- 同类样本打分是否稳定
- 证据不足样本是否被谨慎处理

### 4.5 API 质量

- 结构稳定性
- 错误响应可解析性
- 配额行为一致性

## 5. 人工抽检策略

第一阶段建议保留人工抽检机制。

抽检重点：

- 每日新增文档摘要
- 高分机会项
- 存在冲突的主题
- watchlist 命中的内容
- Agent API 的关键响应

## 6. 上线前最低测试门槛

第一阶段上线前至少满足：

- 核心规则层具备基础单元测试
- 数据闭环具备集成测试样本
- 日报模板有稳定输出样本
- 机会评分有固定案例回归测试
- API 合约字段冻结并通过验证

## 7. 后续扩展

后续可增加：

- 更系统的回归测试集
- 更细粒度质量评分仪表盘
- 更强的人工标注对照集
- 自动化质量报警
