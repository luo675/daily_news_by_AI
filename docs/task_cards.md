# 任务卡

## 1. 使用说明

本文件用于把第一阶段规划拆成可直接分配给单独模型执行的任务卡。

每张任务卡包含：

- 任务目标
- 范围边界
- 输入
- 输出
- 依赖
- 验收标准
- 建议交付物
- 适合的模型类型

设计原则：

- 单任务尽量独立
- 单任务输出应可被后续任务直接消费
- 单任务避免跨越多个大模块
- 优先产出文档、规范、接口草案和结构定义
