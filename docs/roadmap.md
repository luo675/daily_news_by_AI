# 里程碑规划

## 1. 规划原则

- 先定边界，再做实现
- 先打通闭环，再做增强
- 先支持个人使用，再考虑更复杂协作
- 每个阶段都要有明确可验收交付物

## 2. 里程碑总览

### M1 设计定稿

目标：

- 完成项目定义
- 完成数据模型
- 完成 API 草案
- 完成评分与日报模板

输入：

- `goal.md`
- 当前确认的项目决策

输出：

- 项目总览文档
- 架构文档
- 数据模型草案
- API 草案
- 任务拆分

验收标准：

- 架构边界清晰
- 第一阶段范围明确
- API、数据模型、评分规则不冲突

### M2 数据闭环

目标：

- 打通来源接入、处理、入库闭环

输入：

- 来源规则
- 数据模型

输出：

- 来源注册方案
- 文档生命周期方案
- 采集接口规范
- 处理流水线方案

验收标准：

- 博客、访谈、文字稿三类来源能映射到统一文档对象
- 处理结果可进入存储与索引层

### M3 检索与日报

目标：

- 完成基础检索能力
- 完成中英双语日报模板
- 支持固定日报和按需简报

输入：

- 已入库的结构化文档
- watchlist 规则

输出：

- 检索接口草案
- 日报模板与内容结构
- 报告生成任务流

验收标准：

- 可以围绕主题或关注项返回结构化结果
- 日报结构固定且可稳定复用

### M4 机会判断与 API

目标：

- 完成产品机会评分框架
- 完成面向通用 AI Agent 的结构化 API

输入：

- 检索结果
- 冲突处理规则
- 评分维度与权重

输出：

- 机会判断模板
- 结构化 API 返回格式
- 鉴权与配额设计

验收标准：

- API 能返回机会、风险、不确定性和证据
- 评分结果具备固定维度和统一口径

### M5 人工修订与运维

目标：

- 完成人工修订入口
- 完成质量评估体系
- 完成后续自动化方向规划

输入：

- 前四阶段设计结果

输出：

- 修订字段规则
- 审计记录模型
- 质量指标体系
- 调度与监控策略

验收标准：

- 人工修订可覆盖自动结论
- 质量评估指标可指导后续优化

## 3. 关键依赖

- M2 依赖 M1
- M3 依赖 M2
- M4 依赖 M3
- M5 依赖 M4

并行空间：

- 数据模型设计与来源规则设计可并行
- API 草案与日报模板可在中后期并行完善
- 测试策略可随 M3-M5 同步推进

## 4. 推荐执行顺序

1. 完成文档与边界定义
2. 完成数据模型与来源规则
3. 完成处理流水线设计
4. 完成检索、日报和评分设计
5. 完成 API、鉴权、人工修订与测试策略

## 5. 第一阶段 MVP 收口

第一阶段 MVP 应至少保证：

- 可以持续接入英文优先的高价值文字内容
- 可以生成摘要、主题、实体和机会候选项
- 可以生成结构化中英双语日报
- 可以输出产品机会评分
- 可以给通用 AI Agent 提供结构化 API
- 可以进行人工修订

不要求第一阶段解决：

- 自动化转录全链路
- 复杂知识图谱
- 多人协作
- 公开内容平台
